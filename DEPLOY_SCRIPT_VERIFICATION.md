## 2. Цілісність завантаження (Download Integrity)

Для запобігання пошкодженню файлів під час завантаження через мобільні мережі (ризик обриву зв'язку для файлу 4.4 ГБ вкрай високий) впроваджено функцію `check_and_download`:

1.  **Докачування (Resume)**:
    Використовується прапорець `curl -L -C -`. Якщо завантаження переривається, при повторному запуску або запиті на повтор `curl` продовжить запис з того місця, де зупинився, замість повторного скачування з нуля.
2.  **Валідація розміру (Size Verification)**:
    Після завершення `curl` скрипт перевіряє розмір файлу за допомогою `stat` (з підтримкою синтаксису Linux/Termux `stat -c%s` та macOS/BSD `stat -f%z`):
    *   Для моделі перекладу: очікується рівно **`4,624,650,016` байт**.
    *   Для архіву Supertonic 3: очікується рівно **`128,774,318` байт**.
3.  **Повідомлення про помилки**:
    Якщо розмір файлу не збігається, скрипт виводить чітке попередження червоним кольором (`[ERROR] Download of... was incomplete`) та запитує користувача: `Do you want to resume/retry the download? (Y/n)`. Це запобігає використанню бітих моделей.

## 3. Тестування у чистому середовищі (Clean Environment Test)

Було змодельовано встановлення у порожній директорії без попередніх конфігів та моделей:

1.  **Парсинг аргументів**:
    Команда `bash deploy_oneplus13.sh --help` коректно повертає використання.
2.  **Діалоговий режим моделей**:
    *   Скрипт розпізнає відсутність `Hy-MT2-7B-Q4_K_M.gguf`. За запитом скачування завантаження починається правильно. Примусове переривання завантаження (Ctrl+C) та повторний запуск успішно викликають докачування (`curl -C -`) і відновлюють прогрес.
    *   Скрипт успішно знаходить відсутність Supertonic 3 моделі, скачує `.tar.bz2` архів, перевіряє його розмір, розпаковує у `models/` та видаляє тимчасовий архів.
3.  **Залежності Ubuntu**:
    Скрипт коректно генерує `ubuntu_setup.sh`, який тепер містить:
    ```bash
    pip install marker-pdf pydantic transformers manga-ocr mokuro pytesseract stress-uk num2words --break-system-packages
    ```
    Встановлення цих пакетів гарантує роботу модуля `NeuralStressifier` без ручного втручання.
4.  **Результат**:
    Скрипт успішно доходить до кінця, налаштовує `~/.bashrc` для автозапуску та виводить підсумкове привітання. Веб-інтерфейс на порту 5000 піднімається і працює.

## 6. Верифікація експорту Манги у формат Kindle (TASK-16)

Для перевірки можливості експорту манги у формат, сумісний з Kindle (`.azw3`), було проведено успішні тести безпосередньо на OnePlus 13:

1. **Інтеграція Mapaki**:
   Скомпільовано нативний бінарний файл `mapaki` на архітектурі ARM64 всередині контейнера PRoot Ubuntu.
2. **Даунскейл зображень**:
   Інтегровано Pillow-обробник у [translate_manga.py](file:///home/vokov/projects/kindle-butch-gen/translate_manga.py). Всі зображення з висотою понад 1920px (наприклад, оригінальні 1500x2250) даунскейляться до **1280x1920** (зберігши співвідношення сторін). Це запобігає багу пустих сторінок на читалках Kindle Scribe.
3. **Запуск тестової конвертації манги**:
   - Чекбокс **Build Kindle AZW3** виставлено активним.
   - Тестування проведено на манзі `frieren` (194 сторінки).
   - Лог запуску підтверджує успішну обробку всіх сторінок:
     ```text
     [translate_manga.py] Preprocessing translated images for Kindle (downscale height > 1920px)...
     [translate_manga.py] Downscaling ...png from 1500x2250 to 1280x1920
     [translate_manga.py] Packaging pages into CBZ archive...
     [translate_manga.py] Generating AZW3 using Mapaki...
     [translate_manga.py] Running Mapaki: /root/go/bin/mapaki -i /tmp/tmp_out -o .../frieren_translated_uk.azw3 --title "Frieren Translated Uk"
     Processing images 194 / 194 [--------------------------------------] 100.00% 134 p/s
     [translate_manga.py] AZW3 manga generated successfully at: .../frieren_translated_uk.azw3
     [translate_manga.py] Manga translation completed successfully!
     ```
   - У результаті успішно сформовано два файли:
     *   `frieren_translated_uk.cbz` — **`228 MB`** (архів оригінального розміру)
     *   `frieren_translated_uk.azw3` — **`108 MB`** (Kindle-сумісна версія з даунскейлом до 1280x1920)
4. **Контроль опцій з дашборду**:
   - Перевірено роботу додаткових чекбоксів на дашборді (**Clean Pages**, **Translate**, **Build Kindle AZW3**).
   - Завдяки прапорцю `--no-translate` користувач може пропускати стадію OCR-перекладу і миттєво упаковувати наявний кеш картинок у нову збірку AZW3.
   - Оновлені файли завантажень автоматично з'являються у списку скачування книжкової картки на дашборді.

## 7. Адаптація роздільної здатності під різні пристрої Kindle (TASK-16)

Спеціально для оптимізації рендерингу та економії місця на дисках рідерів додано можливість гнучкого вибору цільової роздільної здатності екрана:

1. **Додано селектор роздільної здатності на дашборд**:
   Під чекбоксами манги виведено список вибору пристрою / дозволу:
   *   **Safe Default (1280x1920)** — універсальний безпечний варіант.
   *   **Kindle Scribe (1860x2480)** — максимальна чіткість для 10.2" екрана.
   *   **Paperwhite 5 / Oasis 3 (1236x1648)** — адаптовано під екрани 6.8" / 7".
   *   **Paperwhite 3/4 / Voyage / Basic 11 (1072x1448)** — стандарт для 6" 300 ppi екранів.
   *   **Kindle Basic / Older (600x800)** — мінімальна роздільна здатність для старих читалок (167 ppi).
   *   **Original (No resizing)** — без масштабування картинок.
2. **Передача параметрів на бекенд**:
   Метод `runConversion` зчитує обрану роздільну здатність (`manga_resolution`) та передає її у POST-запиті.
3. **Обробка та даунскейл у `translate_manga.py`**:
   - Аргументи `--max-width` та `--max-height` зчитуються скриптом перекладу манги.
   - Обробник перевіряє межі зображення по обох осях, пропорційно зменшуючи його, якщо воно перевищує бодай один ліміт. Це дозволяє генерувати `.azw3` файли значно меншого розміру для менших екранів, що критично прискорює завантаження книг та заощаджує простір на читалці.
