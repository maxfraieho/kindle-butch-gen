# project-memory: kindle-butch-gen
Last Updated: 2026-07-08T02:45:00Z
Agent: Antigravity (Termux environment)

## 1. Текущий статус проекта
Мы успешно провели рефакторинг и обобщили проект `kindle-butch-gen` для поддержки нескольких книг и гибких настроек TTS:
*   **Голос**: Підтримується преміум-опція TTS Supertonic 3 (Flow Matching, ~99M параметрів) з підтримкою 10 дикторів (0-9) нативного mono-формату (22050 Hz). Двигун Piper та застарілі голоси Piper повністю видалені з інтерфейсу та заблоковані на користь Supertonic 3.
*   **Апаратне прискорення GPU через NNAPI**: Успішно скомпілювали та встановили `sherpa-onnx` із підтримкою Android NNAPI Execution Provider нативно в Termux (обійшовши compile-time обмеження API Level). Реалізовано стабільну гібридну архітектуру:
    *   `duration_predictor`, `text_encoder`, `vector_estimator` — прискорюються на **GPU/NPU** через NNAPI.
    *   `vocoder` (нейромережевий вокодер) — виконується на **CPU** (4 потоки NEON/xnnpack), оскільки його структура викликала збої (segfault) у графічному драйвері Adreno поза стандартною JVM Android.
*   **Оптимізація та якість**:
    *   Кількість кроків генерації Flow Matching повернуто до **8** (преміальна якість просодії та інтонацій на вимогу користувача).
    *   Аудіопотік децимується вдвічі (`samples[::2]`) перед записом, знижуючи частоту дискретизації до 22050 Гц. Це скоротило витрати на конвертацію `float -> int16` та дисковий запис (I/O) на **50%**, а также зменшило розмір папки аудіочанків удвічі (~750 МБ замість 1.5 ГБ).
    *   **Фільтрація та шумопоглинання через FFmpeg**: При фінальному злитті чанків додано професійний ланцюжок фільтрів мовлення: `-af "afftdn,highpass=f=80,lowpass=f=8000,speechnorm"`. Він виконує інтелектуальний FFT-денойз, зрізає інфразвукові гули/ультразвукові шуми та динамічно вирівнює рівні гучності для ефекту студійного запису.
*   **Стійкість до переривань (Резильєнтність)**:
    *   **Динамічне кешування**: У `bin/tts_helper.py` додано моментальний запис чанків у файл кешу `tts_cache_supertonic-3-tts-int8.json` відразу після успішної генерації кожного окремого фрагмента. У разі припинення процесу прогрес не втрачається.
    *   **Кеш наголосів (Stress Cache)**: У `audio_stage.py` реалізовано кешування наголосів `stress_cache_uk.json`. Пакетний наголошувач `Stressifier` у контейнері PRoot Ubuntu тепер запускається лише для нових (ще не наголошених) абзаців, що скорочує час повторного старту з 10 хвилин до 1 секунди.
*   **Веб-интерфейс**: Flask-дашборд повністю очищений від налаштувань Piper, вибір рушія видалено. Доступні повзунки швидкості, шумів та вибір спікерів Supertonic 3 (0-9).
*   **Повний депрекейшн Piper**:
    *   Усі бінарні файли та залежності Piper (`bin/piper/`) повністю видалені з диска (вивільнено **51 МБ**).
    *   Моделі Piper (`models/piper/`) видалені з диска (вивільнено **154 МБ**).
    *   Видалено скрипт `bin/piper_helper.py`.
    *   Гілку `master` на dev-сервері успішно зв'язано з `origin/master`. **ВАЖЛИВО**: при rsync-синхронізації обов'язково виключати каталог `.git/` (`--exclude=".git/"`), щоб локальний Termux-конфіг (у якому немає налаштованого origin/master через відсутність SSH-ключів) не перезаписував налаштування гілок на dev-сервері.

## 2. Последние измененные файлы
*   `bin/tts_helper.py` (гібридне апаратне прискорення NNAPI, децимація до 22050 Hz, num_steps=8, динамічне кешування, видалено `run_piper`)
*   `audio_stage.py` (інтеграція майстер-кешу наголосів `stress_cache_uk.json`, захист від зависання через stdin DEVNULL, видалено залежність від Piper, додано фільтри FFmpeg)
*   `bin/stressify_batch.py` (пакетний наголошувач та NFD-нормалізатор для запуску в PRoot Ubuntu)
*   `kbg_web/app.py` (повне видалення Piper з інтерфейсу, фіксація налаштувань та дашборду на Supertonic 3)
*   `books/vibe-programming/config.json` (оновлено налаштування під Supertonic 3)
*   `common/book_paths.py` (змінено дефолтний рушій на `supertonic3`)
*   `README.md` (оновлено опис проекту під Supertonic 3)
*   `bin/piper_helper.py` (видалено)
