# Система підтримки: два канали + реєстрація + реферали (P3)

> Специфікація від Q, 2026-07-17. Джерело правди для цієї фічі.
> Статус: Phase 1 в реалізації; Phase 2-3 заплановані.

## Специфікація (від Q, дослівно за змістом)

### Два окремі канали — ніколи не змішувати
- **Track A — верифікований фонд**: офіційна ЗАГАЛЬНА сторінка донату
  «Повернись живим» (savelife.in.ua) або UNITED24 (обидва перевірені, активні,
  публічна звітність станом на липень 2026). НЕ конкретна кампанія (збори
  завершуються — хардкоджений URL застаріє). Підпис у банері: "офіційний фонд,
  публічна звітність". Розробник грошей цього треку не торкається взагалі.
- **Track B — підтримка автора інструменту**: monobank-банка / BMC / Patreon,
  підписано явно "підтримати розробника цього інструменту", НЕ як воєнний збір.
  Людина завжди бачить два різні призначення і вирішує сама.

### Механіка вставки
- Інтервал: випадкове 50–70 сторінок від останньої вставки.
- Точка: найближча природна пауза — кінець розділу (EPUB-структура; для манги
  кінець розділу/арки або епізоду).
- Сюжетна прив'язка: легкий перехід без спойлерів ("Поки [персонаж] переводить
  подих..."); евристика тону — НЕ вставляти одразу після емоційно важкої сцени.
- Формат — візуально/акустично відокремлений від авторського тексту:
  - EPUB: окремий блок, свій CSS-клас (рамка/шрифт/фон).
  - Аудіо: інший голос/тон або звукова позначка + слова "Коротка примітка від
    сервісу, не від автора книги."
  - Манга: окрема інтерстиційна сторінка, інший візуальний шаблон.

### Opt-out — реальний, одним кроком
- Бот: `/no_support_banner` → миттєво прапорець у профілі → "Готово".
- Або перемикач у налаштуваннях на сайті. Ніколи не ховати і не ускладнювати.

### Реєстрація
- Бот: `/start` → короткий опис → токен доступу, прив'язаний до Telegram ID.
- Сайт (1 сторінка): що це, як почати (лінк на бота), прозоро про Track A/B,
  лінк на open-source репо. Без SEO-лендінгу і агресивної афішації.

### Реферали
- `/referral` → персональний код; новий користувач за кодом → бонус обом.
- Бонуси функціональні, не грошові: пріоритет у черзі генерації, більше
  одночасних книг, швидша генерація. Без платіжної системи.

## Архітектурні рішення

1. **kbg сьогодні однокористувацький** (один Flask на пристрої, basic auth).
   Мультикористувацькі частини (бот, токени, профілі, реферали, сайт) — це
   ОКРЕМИЙ сервіс (dev-сервер 192.168.3.184 або хмара), НЕ код на телефоні.
   Пайплайн на пристрої лише читає конфіг підтримки і поважає opt-out.
2. **Конфіг, не хардкод:** `support_config.json` у корені репо — URL Track A
   (загальні сторінки фондів), URL Track B, тексти банерів, enabled/interval.
   Пайплайн читає його на старті; відсутність файлу або `"enabled": false`
   = жодних вставок (safe default для локальних користувачів).
3. **Опт-аут локально:** поле `no_support_banner` у `global_settings.json`
   (телефонний користувач) — той самий прапорець, який пізніше синхронізує бот.

## Фази

### Phase 1 — вставки в пайплайні (телефонний код, ця сесія)
- [ ] `support_config.json` + читання в common/ (одна функція, safe defaults).
- [ ] EPUB: інтерстиційний блок з окремим CSS-класом у кінці розділу,
      лічильник сторінок 50–70 (випадковий, seeded від slug для відтворюваності).
- [ ] Манга: інтерстиційна PNG-сторінка (генерується з шаблону PIL) у CBZ
      на межі розділу/епізоду.
- [ ] Аудіо: вставка озвученої примітки іншим голосом/позначкою на межі розділу.
- [ ] Евристика тону (ключові слова останнього уривка) — skip якщо важка сцена.
- [ ] Повага до `no_support_banner` у global_settings.json.

### Phase 2 — Telegram-бот + профілі (dev-сервер)
- [ ] Бот: /start (токен ↔ TG ID), /no_support_banner, /referral.
- [ ] Сховище профілів (SQLite на dev-сервері).
- [ ] Синхронізація прапорця opt-out і бонусів із пайплайном.

### Phase 3 — сайт (1 сторінка) + реферальні бонуси в черзі
- [ ] Статична сторінка (Cloudflare Pages або dev-сервер NGINX).
- [ ] Пріоритет у черзі генерації для рефералів (інтеграція з наявною чергою).

## Обмеження/ризики
- Тексти банерів українською; жодних спойлерів; тон-евристика консервативна
  (false positive "важкої сцени" = просто пропустити вставку, не страшно).
- Track A URL — тільки кореневі сторінки фондів: `https://savelife.in.ua/donate/`,
  `https://u24.gov.ua/` (перевірити перед релізом Phase 1).
- Юридично: Track B підписаний як підтримка розробника інструменту.
